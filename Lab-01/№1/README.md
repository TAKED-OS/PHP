## 2.Метод запроса:GET 

Почему именно данный метод используется для загрузки страницы?

GET-специально предназначен для извлечения данных с сервера без изменения чего-то на нём-просто "дай мне эту страницу"

## 3.Статус ответа:304 Not Modified

Что означает статус ответа?

304 Not Modified-Когда браузер уже один раз загрузил страницу или файл, он сохраняет его у себя в кэше.При следующем запросе он спрашивает сервер если данная версия является актуальной или есть новая версия. Если сервер проверяет и понимает что файл остался без изменений то отвечает 304 Not Modified.
Что при это делает браузер:

  не скачивает файл повторно

  берет из своего кэша

  страница загружается быстрее 
  экономится трафик

## 4.Заголовки запроса и ответа. Какие заголовки присутствуют и какую информацию они содержат?

Присутствуют заголовки Request Headers и Response Headers

Что они содержат:

  ### Request Headers:

    Основные (HTTP/2 псевдо-заголовки):

      :method — метод запроса (GET)

      authority — доменное имя сервера (en.wikipedia.org)

      :path — путь к ресурсу (/wiki/HTTP)

      :scheme — протокол (https)

    Информация о поддерживаемых форматах:

      Accept — типы данных, которые браузер может принять (HTML, XML, изображения и др.)

      Accept-Encoding — поддерживаемые методы сжатия (gzip, br, zstd)

      Accept-Language — предпочтительные языки (ru, en, ro)
    Кэширование:

      Cache-Control — управление кэшированием (max-age=0 — требуется проверка актуальности)

      If-Modified-Since — дата последней версии ресурса, имеющейся у клиента
    Дополнительная информация:

      Cookie — данные сессии и пользовательская информация

      Referer — источник перехода

      User-Agent — информация о браузере и ОС

      Sec-* заголовки — параметры безопасности и контекста запроса

  ### Response Headers:

    Информация о содержимом:

      Content-Type — тип данных (text/html; charset=UTF-8)

      Content-Encoding — способ сжатия (gzip)

      Content-Language — язык контента (en)

    Кэширование:

      Cache-Control — правила хранения в кэше

      Age — время хранения ресурса в CDN-кэше

      Vary — какие заголовки влияют на кэширование

      Last-Modified — дата последнего изменения ресурса

    Безопасность:

      Strict-Transport-Security — принудительное использование HTTPS

      Content-Security-Policy — политика загрузки ресурсов

      X-Content-Type-Options — запрет на определение типа файла по содержимому

    Техническая информация:

      Server — имя сервера

      Date — дата ответа

      X-Request-Id — уникальный идентификатор запроса

      X-Cache / X-Cache-Status — информация о кэшировании на стороне CDN

      NEL, Report-To — параметры мониторинга сетевых ошибок



      
## 5.Есть ли тело запроса или ответа? Если да, то что в нем содержится?

Тело запроса-его нет потому что метод-GET. GET-обычно не содержит тело,весь запрос состоит из строки запроса+заголовков

Тело ответа-его тоже нет потому что статус 304 Not Modified, А 304 по спецификации не должен содержать тело.

## 6.Какие еще запросы были отправлены при загрузке страницы и почему?

При загрузке страницы браузер сначала отправляет основной GET-запрос для получения HTML-документа. После получения HTML браузер анализирует его содержимое и обнаруживает ссылки на дополнительные ресурсы.В результате отправляются дополнительные HTTP-запросы для загрузки CSS-файлов (стили страницы), JavaScript-файлов (интерактивность), изображений и иконок. Каждый ресурс загружается отдельным запросом.Таким образом, дополнительные запросы необходимы для корректного отображения и функционирования веб-страницы.

  ### URL запроса. Какой URL используется для выполнения поиска?

  ### Метод запроса. Почему используется именно этот метод?
  Метод запроса-GET. Используется метод GET Потому что это получение данных, поиск ничего не изменяет на сервере, он просто запрашивает информацию.А по стандарту HTTP:GET-получение данных
  
  ### Query Parameters. Какие параметры передаются в запросе, что они означают и для чего используются?

  В поисковом запросе к ресурсу /w/index.php используются Query Parameters, которые передаются в URL после символа ?. В данном запросе присутствуют следующие параметры:
    
    1. search=browser- Передаёт поисковый запрос пользователя.Используется для определения слова или фразы, по которым выполняется поиск.

    3. advancedSearch-current={...}- Содержит параметры расширенного поиска (например, поиск в заголовке статьи).Используется для уточнения условий поиска.

## URL запрос:

## Метод запроса: GET

## Статус ответа: 

200 OK (from service worker)- это означает что ответ успешно получен,но его вернул не сервер напрямую,а service worker. Service Worker это специальный скрипт в браузере, который:

    перехватывает HTTP-запросы

    может отдавать данные из кэша

    может работать офлайн

    управляет кэшированием

## Заголовки запроса и ответа:

  ### Заголовки запроса (Request Headers)

  В запросе присутствуют заголовки, передающие серверу информацию о клиенте и параметрах запроса:

    User-Agent — содержит информацию о браузере и операционной системе пользователя.

    Referer — указывает источник перехода на страницу.

    Device-Memory — объём оперативной памяти устройства.

    Sec-CH-DPR, Sec-CH-Viewport-Width — параметры экрана (плотность пикселей и ширина viewport).

    Sec-CH-UA, Sec-CH-UA-Platform, Sec-CH-UA-Mobile — сведения о браузере, платформе и типе устройства.

    Upgrade-Insecure-Requests — указывает на необходимость использования HTTPS.

  Эти заголовки используются для идентификации клиента и адаптации контента под устройство пользователя.

  ### Заголовки ответа (Response Headers)

  В ответе сервера присутствуют заголовки, содержащие информацию о ресурсе и правилах его обработки:

    Content-Type — тип передаваемого ресурса (HTML-документ).

    Content-Encoding — способ сжатия данных (Brotli).

    Cache-Control, Pragma, Expires — правила кэширования (запрет хранения в кэше).

    Content-Security-Policy — политика безопасности загрузки ресурсов.

    Strict-Transport-Security — принудительное использование HTTPS.
  
    X-Frame-Options — запрет встраивания страницы в iframe с других сайтов.

    X-Content-Type-Options — запрет определения типа файла по содержимому.

    Alt-Svc — информация о поддержке альтернативных протоколов (например, HTTP/3).

## Тела запроса и ответа:

  ### Тело запроса

  В данном случае тело запроса отсутствует, так как используется метод GET.Метод GET предназначен для получения данных и обычно не содержит тела — вся необходимая информация передаётся в строке запроса и заголовках.

  ### Тело ответа

  Тело ответа присутствует, так как сервер вернул статус 200 OK.В теле ответа содержится HTML-документ страницы YouTube, который затем обрабатывается браузером. После этого браузер дополнительно загружает связанные ресурсы (CSS, JavaScript, изображения).
