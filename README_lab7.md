# Лабораторная работа №7. Шаблонизация

## Тема

Шаблонизация в PHP. Разделение логики приложения и представления с использованием нативных PHP-шаблонов и шаблонизатора Twig.

---

## Цель работы

Освоить принципы шаблонизации в PHP, научиться отделять логику обработки данных от отображения интерфейса, а также подключить и использовать готовый шаблонизатор Twig через Composer.

---

## Описание проекта

В рамках лабораторной работы был продолжен проект из лабораторной работы №6 — **дневник настроения**.

Проект позволяет пользователю добавлять записи о своём состоянии за день. Для каждой записи можно указать:

- название записи;
- описание дня;
- настроение;
- причину или событие, которое повлияло на настроение;
- дату записи;
- количество часов сна;
- был ли день социальным.

Данные сохраняются в файл `data.txt` в формате JSON.

Главная задача лабораторной работы — не изменение внешнего вида сайта, а изменение внутренней структуры проекта. Код был разделён на два слоя:

1. **Логика приложения** — обработка данных, валидация, чтение и сохранение записей.
2. **Представление** — отдельные шаблоны, отвечающие только за вывод HTML.

В проекте реализованы два варианта отображения:

1. Через нативные PHP-шаблоны.
2. Через шаблонизатор Twig.

## Назначение основных файлов

### `index.php`

Главная страница проекта с использованием нативных PHP-шаблонов.

Файл загружает данные, получает ошибки из сессии и подключает шаблоны из папки `templates`.

---

### `twig.php`

Страница проекта с использованием шаблонизатора Twig.

Файл подключает Composer, настраивает Twig, передаёт данные в Twig-шаблоны и регистрирует пользовательский фильтр `mood_icon`.

---

### `process.php`

Файл обработки формы.

Он принимает данные методом `POST`, проверяет их через функцию валидации и сохраняет новую запись в файл `data.txt`.

---

### `src/functions.php`

Файл с основной логикой проекта.

В нём находятся функции для:

- загрузки данных;
- сохранения данных;
- добавления новой записи;
- валидации формы;
- преобразования настроения в иконку.

---

### `templates/`

Папка с нативными PHP-шаблонами.

Эти файлы отвечают только за отображение данных:

- `layout.php` — общий макет страницы;
- `form.php` — форма добавления записи;
- `list.php` — таблица со списком записей.

---

### `twig_templates/`

Папка с Twig-шаблонами.

В ней находятся:

- `layout.twig` — общий макет страницы;
- `form.twig` — форма добавления записи;
- `list.twig` — список записей.

## Реализация нативных PHP-шаблонов

На первом этапе лабораторной работы были созданы обычные PHP-шаблоны.

Пример вывода данных в PHP-шаблоне:

```php
<?= htmlspecialchars($entry['title']) ?>
```

В этом варианте PHP используется прямо внутри HTML, но логика обработки данных вынесена в отдельные файлы.

Преимущество такого подхода в том, что он не требует дополнительных библиотек и работает сразу в PHP.

Недостаток — при росте проекта PHP-код и HTML могут снова начать смешиваться, если не следить за структурой.

---

## Реализация Twig-шаблонов

На втором этапе был установлен и подключён шаблонизатор Twig.

Twig-шаблоны находятся в папке:

```text
twig_templates/
```

Пример вывода данных в Twig:

```twig
{{ entry.title }}
```

Также в проекте используется наследование шаблонов.

Файл `form.twig` наследует общий макет:

```twig
{% extends "layout.twig" %}
```

Контент страницы вставляется в блок:

```twig
{% block content %}
    ...
{% endblock %}
```

Это позволяет не повторять один и тот же HTML-каркас на каждой странице.

---

## Дополнительная функция

Для получения максимальной оценки был создан собственный фильтр Twig:

```php
$twig->addFilter(new \Twig\TwigFilter('mood_icon', function ($mood) {
    return moodIcon($mood);
}));
```

Фильтр используется в шаблоне так:

```twig
{{ entry.mood|mood_icon }}
```

Он преобразует текстовое значение настроения в эмодзи.

Например:

```text
happy  → 😊
normal → 😐
sad    → 😔
angry  → 😡
tired  → 😴
```

Это практическое улучшение, потому что список записей становится более наглядным для пользователя.

## Сравнение двух подходов

### Нативные PHP-шаблоны

Преимущества:

- не требуют установки дополнительных библиотек;
- просты для маленьких проектов;
- используют обычный PHP-синтаксис.

Недостатки:

- легко смешать логику и HTML;
- нужно вручную использовать `htmlspecialchars`;
- при росте проекта код становится сложнее поддерживать.

---

### Twig

Преимущества:

- шаблоны выглядят чище;
- есть наследование шаблонов;
- есть блоки, include и фильтры;
- проще отделять логику от представления;
- Twig автоматически экранирует вывод.

Недостатки:

- нужно устанавливать через Composer;
- нужно изучить отдельный синтаксис;
- для очень маленьких проектов может быть избыточным.

## Ответы на контрольные вопросы

### 1. В чём отличие нативных PHP-шаблонов от шаблонизатора Twig? Какие преимущества и недостатки у каждого подхода?

Нативные PHP-шаблоны — это обычные `.php` файлы, в которых HTML совмещается с небольшими PHP-вставками. Например, для вывода переменной используется `<?= htmlspecialchars($title) ?>`.

Twig — это отдельный шаблонизатор со своим синтаксисом. В нём данные выводятся через двойные фигурные скобки, например `{{ title }}`.

Преимущество нативных PHP-шаблонов в том, что они не требуют установки дополнительных библиотек. Они простые и подходят для небольших проектов.

Недостаток в том, что при неправильной организации код может быстро стать грязным, потому что HTML и PHP-логика начинают смешиваться.

Преимущество Twig в том, что он делает шаблоны чище и удобнее. В нём есть наследование шаблонов, блоки, подключение других шаблонов и фильтры.

Недостаток Twig в том, что его нужно устанавливать через Composer и изучать отдельный синтаксис.

---

### 2. Зачем разделять логику и представление в проекте? Какие проблемы могут возникнуть, если смешивать их в одном файле?

Разделение логики и представления нужно для того, чтобы проект был понятнее и удобнее для поддержки.

Логика отвечает за обработку данных: получение формы, проверку, сохранение и загрузку записей.

Представление отвечает за внешний вид: HTML, таблицы, формы и вывод данных пользователю.

Если смешивать всё в одном файле, код становится сложным для чтения. Также становится сложнее искать ошибки, менять дизайн и добавлять новые функции. При изменении HTML можно случайно сломать обработку данных.

Поэтому в проекте логика была вынесена в `src/functions.php` и `process.php`, а отображение — в отдельные шаблоны.

---

### 3. Что такое наследование шаблонов в Twig? Как работают `{% extends %}` и `{% block %}`?

Наследование шаблонов в Twig позволяет создать один общий макет страницы и использовать его для разных страниц проекта.

Например, в `layout.twig` находится общий HTML-каркас сайта: `html`, `head`, меню, контейнер и другие общие элементы.

Команда:

```twig
{% extends "layout.twig" %}
```

означает, что текущий шаблон наследует общий макет.

Команда:

```twig
{% block content %}
    ...
{% endblock %}
```

создаёт участок, куда вставляется содержимое конкретной страницы.

Таким образом, общий дизайн хранится в одном файле, а разные страницы заполняют только нужные блоки.

Это удобно, потому что не нужно копировать одинаковый HTML-код в каждый шаблон.
